# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do not** open a public issue.
2. Email **security@tiagoandrepro.com.br** with a description of the vulnerability.
3. Include steps to reproduce and potential impact.

We will acknowledge receipt within 48 hours and aim to release a fix within 7 days for critical issues.

## Scope

- API key authentication and authorization
- Rate limiting bypass
- Data exposure or injection
- Server-side request forgery

## Out of Scope

- Meta's developer documentation content (report to Meta directly)
- Denial-of-service via high request volume (rate limiting is in place)
